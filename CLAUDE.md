## Architecture

This is an **Astro 5 static blog** with Tailwind CSS 4. There is no backend or database — all content is pre-compiled at build time.

### Content Collections

Blog posts are Markdown files in `src/posts/` managed via Astro Content Collections. The schema is defined in `src/content.config.ts`:

```ts
// Required frontmatter for all posts
title: string
date: Date
tags: string[]
description?: string  // optional, used for OG meta tags
```

Posts also include `layout: "@layouts/Layout.astro"` in frontmatter to apply the main layout directly.

### Routing

File-based routing under `src/pages/`:
- `/` — Homepage showing 4 most recent posts + full tag list
- `/blog/page/[page]` — Paginated archive (8 posts per page)
- `/blog/[slug]` — Individual post pages (slug = filename without `.md`)
- `/tag/[tag]` — Posts filtered by tag
- `/rss.xml` — RSS feed (generated by `src/pages/rss.xml.ts`)
- `/login`, `/signup`, `/pricing` — Exist but are hidden from navigation

### Styling

Tailwind 4 is configured via the `@tailwindcss/vite` plugin in `astro.config.mjs` — there is no `tailwind.config.*` file. Global styles and `@layer base` markdown element overrides live in `src/styles/global.css`. The responsive navbar (hamburger on mobile, horizontal at 700px+) is driven by `src/main.ts` with corresponding styles in `global.css`.

### Path Aliases (tsconfig.json)

- `@/*` → project root
- `@layouts/*` → `src/layouts/*`

### Integrations

- `@astrojs/sitemap` — auto-generates sitemap at build time (configured site: `https://mygreatsite.com`)
- `@astrojs/rss` — RSS feed at `/rss.xml`
- Astro ClientRouter — view transitions between pages
- Font Awesome 5.14.0 — loaded via CDN in `src/layouts/Layout.astro`
